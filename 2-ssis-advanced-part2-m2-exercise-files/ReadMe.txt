The demonstrations in this module assumes that you have installed AdventureWorksDW2012 database in a local SQL Server 2014 database instance. It also requires a Windows Azure subscription, which you may acquire if you have an active MSDN subscription or you may sign up for a free trial at http://azure.microsoft.com/en-us/pricing/free-trial/. 

1. You must install a database engine instance of SQL Server 2014 (any edition), SQL Server Management Studio, SQL Server Data Tools for Visual Studio 2012, and Integration Services. 

2. This demonstration assumes that you have installed AdventureWorks2012 database in the local SQL Server 2012 database instance. You can download the database available from Codeplex, http://msftdbprodsamples.codeplex.com, labeled SQL Server 2012 OLTP. Installation instructions are at  http://social.technet.microsoft.com/wiki/contents/articles/3735.sql-server-samples-readme-en-us.aspx#Readme_for_Adventure_Works_Sample_Databases.

3. After creating a virtual machine in the demonstration for this module, you must install additional software - Microsoft 
SQL Server Data Tools for Business Intelligence Visual Studio 2012 - which you can download from http://www.microsoft.com/en-us/download/details.aspx?id=36843. 

4. To create a self-signed certificate to use with a point-to-site VPN after creating a virtual network in the demonstration for this module, you can use Visual Studio 2013, which you can download from https://msdn.microsoft.com/en-us/library/dd831853.aspx.

5. To work with the packages to compress files to/from Azure, you must download and install AZCopy on your own computer and on the Azure virtual machine. You can download AZCopy from http://aka.ms/downloadazcopy. 

6. Extract the ssis-adv-data-2.zip file to the C:\ drive.
